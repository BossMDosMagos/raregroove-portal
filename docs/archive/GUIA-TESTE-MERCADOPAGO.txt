```
[PaymentSuccess] Pagamento aprovado! Verificar transação existente...
[PaymentSuccess] Verificando status do item...
[PaymentSuccess] ✅ Status do item corrigido! (se necessário)
```

**No banco (Supabase SQL Editor):**
```sql
-- Ver o item vendido
SELECT id, title, is_sold, status 
FROM items 
WHERE title ILIKE '%TESTE MP%'
ORDER BY created_at DESC 
LIMIT 1;

-- Deve retornar:
-- is_sold = true
-- status = 'vendido'

-- Ver a transação
SELECT id, price, status, payment_method, created_at
FROM transactions
WHERE item_id IN (
  SELECT id FROM items WHERE title ILIKE '%TESTE MP%'
)
ORDER BY created_at DESC
LIMIT 1;

-- Deve ter status = 'pago' ou 'concluido'
```

## 🐛 SE DER ERRO

### Erro: "Access token do Mercado Pago não informado"
**Solução:**
```sql
-- Verificar se as credenciais estão preenchidas
SELECT mp_public_key_sandbox, mp_access_token_sandbox 
FROM platform_settings WHERE id = 1;

-- Se vazias, preencha no admin: http://localhost:5173/profile
-- Ou diretamente no SQL (substituir pelos seus valores reais):
UPDATE platform_settings 
SET 
  mp_public_key_sandbox = 'TEST-xxx',
  mp_access_token_sandbox = 'TEST-xxx'
WHERE id = 1;
```

### Erro: Item continua no catálogo após venda
**Solução:**
Agora deve ser corrigido automaticamente! Mas se ainda acontecer:
```sql
-- Executar correção manual
UPDATE items 
SET status = 'vendido' 
WHERE is_sold = true AND status != 'vendido';
```

### Erro: "Erro ao criar preferência"
**Solução:**
1. Verificar se Edge Function foi deployada: `npx supabase functions deploy mp-create-preference --no-verify-jwt --use-api`
2. Ver logs no Supabase Dashboard → Functions → mp-create-preference

## 💡 DIFERENÇA ENTRE MP E STRIPE/PAYPAL

| Aspecto | Mercado Pago | Stripe/PayPal |
|---------|--------------|---------------|
| **Checkout** | Abre página externa do MP | Stripe: inline / PayPal: botões |
| **Retorno** | Manual no localhost | Automático |
| **Webhook** | Não configurado | Stripe tem |
| **Status** | Corrigido via PaymentSuccess | Direto via webhook |
